참고 WBS: CRM 연동 SaaS 플랫폼 (실제 완료 프로젝트, 2024)

총 기간: 18주 (당초 계획 14주 → 4주 초과)

Phase 1: 기획 및 설계 (3주, 당초 2주)
- 요구사항 분석: 5일
- DB 스키마 설계: 3일
- API 스펙 정의: 3일
- UI Wireframe: 4일
→ 교훈: 기획에 1주가 더 필요. CRM 연동 스펙 파악에 예상보다 시간 많이 소요

Phase 2: 개발 (9주, 당초 7주)
- BE API 개발: 20일 (당초 15일, 외부 API 연동으로 +5일)
- FE 구현: 18일 (당초 15일, 디자인 변경 2회로 +3일)
- DB 구현: 8일
- 외부 API 연동 (CRM): 12일 (당초 5일, 실제 7일 → 매우 복잡)
→ 교훈: 외부 CRM/서드파티 API 연동은 시간 산정을 2.5배로 적용할 것

Phase 3: 테스트 및 QA (4주, 당초 2주)
- 기능 테스트: 7일
- 통합 테스트: 5일
- 버그 수정: 6일
- 재테스트: 2일
→ 교훈: QA는 전체 기간의 22%가 필요. 버그수정-재테스트 사이클 최소 2회 예상할 것

Phase 4: 배포 (2주)
- 스테이징 배포: 3일
- 클라이언트 UAT: 5일
- 운영 배포: 2일

초과 원인 분석:
1. 서드파티 CRM API 연동 스펙 변경 (2회)
2. 초반 와이어프레임 승인 없이 개발 착수 → 디자인 변경 발생
3. QA Phase 일정 과소 산정
4. 팀원 1명 중간에 2주간 다른 프로젝트 지원으로 투입률 저하
